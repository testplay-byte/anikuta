# Session Log — Session 12 (module structure + package layout + build environment planning)

## Goal
Plan the implementation properly before any code. Cover: single vs multi-module
(with pros/cons for the user to decide), the detailed package layout (what we
copy from aniyomi + our structure), and the build environment (GitHub Actions,
ARM64-v8a only, single APK). NO building.

## What happened
- **minSdk confirmed: 26 (Android 8.0).** Higher-API features (Monet, blur,
  notification permission) gated at runtime — we don't raise minSdk for them.
- **Module structure:** wrote `DOCS/PLAN/MODULE-STRUCTURE.md` explaining single
  vs multi-module (pros/cons). Recommendation: **multi-module (5 modules)** —
  `:app`, `:core`, `:data`, `:domain`, `:source-api`. Drops aniyomi's manga/i18n/
  presentation/benchmark modules. Compiler-enforces UI/logic separation. Maps
  1:1 to aniyomi for upstream tracking. **User to decide.**
- **Package layout:** wrote `DOCS/PLAN/PACKAGE-LAYOUT.md` with:
  - What we copy from aniyomi (source-api anime, domain anime, data anime, core,
    DI wiring, source/extension managers, player, download, backup, trackers
    anime side) → renamed to `app.anikuta.*`.
  - What we do NOT copy (manga-only, aniyomi UI, i18n, presentation, benchmarks,
    archive, metadata, source-local).
  - What we build ourselves (AniList client, Supabase client, caching layer,
    4-design UI, onboarding, navigation).
  - Full proposed package tree (`app.anikuta.*` with all sub-packages).
  - The copy-paste process (find → copy → adapt → document).
  - Module → aniyomi mapping table for upstream tracking.
- **Build environment:** wrote `DOCS/PLAN/BUILD-ENVIRONMENT.md`:
  - GitHub Actions (sandbox has limited RAM/storage — can't build locally).
  - One APK only: `arm64-v8a` (covers ~99% of modern Android phones).
  - minSdk 26, compileSdk latest, JDK 17.
  - Proposed `.github/workflows/build-apk.yml` workflow (checkout → JDK 17 →
    Android SDK → Gradle cache → assembleDebug arm64-v8a → upload artifact).
  - Signing deferred to Phase 8 (release).
  - Sandbox writes code; GitHub Actions builds APK; user installs on phone.
- **Design scope (Phase 1):** Material 3 only, as the fallback design. Other 3
  designs come in Phase 6. All designs fall back to Material 3 if a screen/
  element isn't available in the chosen design.
- Updated `ROADMAP.md`: Phase 0 nearly complete; Phase 1 updated with multi-module
  plan, GitHub Actions, minSdk 26, Material 3 fallback, 7-step onboarding.
- Committed + pushed to GitHub.

## Decisions made
- **Decision 39** — minSdk = 26 (Android 8.0). Higher-API features gated at runtime. — Accepted.
- **Decision 40** — Build environment = GitHub Actions (not sandbox). Single APK, arm64-v8a only. — Accepted.
- **Decision 41** — Material 3 is the fallback design; all designs fall back to it if a screen/element isn't available. — Accepted.
- **Decision 42 (proposed)** — Multi-module (5 modules: :app, :core, :data, :domain, :source-api). — PENDING user decision.
