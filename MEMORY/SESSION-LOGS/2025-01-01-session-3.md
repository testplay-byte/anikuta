## Key findings about aniyomi
- **13 Gradle modules**, clean-architecture layering (i18n → core → source-api
  → domain → data → app). Dependencies only point downward.
- **`:source-api` is the plugin boundary** — 3rd-party extensions compile
  against it; the app loads them at runtime. Most important module to
  understand.
- **Dual anime/manga data model** — aniyomi adds a parallel anime DB
  (`sqldelightanime/`) + anime domain + anime source/UI alongside the manga
  side. Runs through every layer.
- **`:app` is 73% of the code** (952 of ~1,400 kt files). 20 UI screens, 11
  tracker integrations (MAL, AniList, Shikimori, Kitsu, Simkl, Bangumi,
  MangaUpdates, Komga, Kavita, Jellyfin, Suwayomi).
- **DI = Injekt** (not Hilt/Dagger). **Persistence = SQLDelight** (not Room).
  **UI = mixed** View + Compose (migrating to Compose).
