# Session Log — 2026-07-07 (Session 19)

## Goal
Fix the end-to-end video playback pipeline so non-AniKoto extensions actually play. The player loaded for AniKoto but failed for every other extension.

## What happened
Five distinct root causes fixed, each referencing the aniyomi source before deviating:
- **Extension URL crash** (`IllegalArgumentException: Invalid URL host`): extensions only override `videoListRequest`, not `hosterListRequest`. Base class builds `GET(baseUrl + episode.url)` = malformed. Fix: catch ALL `Throwable` in `getHosterList`, fall back to `getVideoList`. (b25c8fa)
- **TLS rejected** (`certificate not correctly signed`): aniyomi ships `cacert.pem`; we don't. Fix: `tls-verify=no`. (2414d12)
- **HTTP 403 Forbidden**: MPV loads the bare URL without the extension's `Referer`/`User-Agent`. Fix: extract `Video.videoHeaders`, pass through intent, set via `MPVLib.setOptionString("http-header-fields", headers)` before `loadfile`. (dbcd967, 1d5f7d2)
- **Infinite loading spinner on failure**: `efEvent` (END_FILE error) not surfaced. Fix: `onFileEnded(errorMessage)` callback → `viewModel.onError()` → ERROR state + "Go Back" button. (dbcd967)
- **Stale header read in Composable**: re-read intent on recomposition. Fix: read once in `onCreate`, pass as parameter. (1d5f7d2)

## Decisions made
- `tls-verify=no` (not ship a CA bundle) — acceptable for a sideloaded app; simpler than maintaining cacert.pem.
- Catch ALL `Throwable` (not just `IllegalStateException`) on `getHosterList` — extension code is varied; a narrow catch masks real failures.
- Surface MPV errors immediately to the user — better UX than an infinite spinner.
