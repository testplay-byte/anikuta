### 2. Applied 5 episode list UX fixes

All 5 issues were reported by the user after testing build `b45431e` on-device:

1. **Metadata enrichment doesn't refresh visible episodes** (NEW bug found by user):
   - **Root cause:** `enrichEpisodesWithMetadata()` mutated `SEpisode` objects in place and set `_episodes.value` with the same list reference. Compose's `LazyColumn` sees the same object references → skips recomposition → enriched fields don't appear until the user scrolls (which disposes and recreates items).
   - **Fix:** Create NEW `SEpisode` objects via `SEpisode.create().apply { ... }` and a new list via `.map { }`. New object references force Compose to recompose visible items immediately.
   - File: `DetailViewModel.kt`

2. **Sub/dub shows HSUB after clicking an episode:**
   - **Root cause:** `getAvailableAudioVersions()` read from the in-memory video cache + persistent PreferenceStore. After clicking an episode, video resolution found "HSUB" and `persistAudioVersions()` saved it. Next render added HSUB to the pills even though the extension's `scanlator` field only said "Sub / Dub".
   - **Fix:** Removed `getAvailableAudioVersions()` and `persistAudioVersions()` methods entirely (and both call sites). Audio detection now uses `SEpisode.scanlator` field only.
   - Files: `DetailViewModel.kt`, `DetailScreen.kt` (removed `availableAudioVersions` parameter from `EpisodeRow`, `EpisodeRowSimple`, `EpisodeRowRich`)

3. **Episodes still in scrollable container when "info above":**
   - **Root cause:** The old code always wrapped episodes in an inner scrollable `LazyColumn` with `heightIn(max=400.dp)`, even when `animeInfoPosition == "above"`.
   - **Fix:** When `animeInfoPosition == "above"` AND episodes are Loaded, episodes render directly as `itemsIndexed()` items in the outer `LazyColumn` (no inner container) — the whole screen scrolls as one long list. Below mode keeps the inner container.
   - File: `DetailScreen.kt` (added `itemsIndexed` import)

4. **No alternating row colors:**
   - **Fix:** Added `index: Int = 0` parameter to `EpisodeRow`. Even-index episodes use `surfaceContainerLow`, odd-index use `surfaceContainerHigh`. Both are standard M3 surface container levels. Inner elements (title/synopsis/thumbnail backgrounds) use `surfaceContainer` (the middle level), so they always contrast with the card background.
   - File: `DetailScreen.kt`

5. **Episodes list too short:**
   - **Fix:** Changed inner `LazyColumn` max height from `400.dp` → `600.dp` (1.5x), applies to below mode.
   - File: `DetailScreen.kt`
