### 5. Dynamic theming from cover image (NEW FEATURE)
- **Palette dependency**: added `androidx.palette:palette-ktx` for color extraction
- **DynamicTheming.kt** (new file):
  - `DynamicColorScheme` data class with 7 colors: primary, surfaceLow, surfaceHigh, surfaceContainer, background, onSurface, onSurfaceVariant
  - `extractDynamicColors()`: loads cover image bitmap via OkHttp (downscaled to 100px for speed), uses Palette API to extract swatches
  - Color extraction priority: vibrant → darkVibrant → muted → darkMuted → lightMuted → dominant (with fallbacks at each step)
  - Darken/lighten extensions for generating surface variants
- **DetailScreen**:
  - `LaunchedEffect` extracts colors when anime loads (if dynamic theming enabled)
  - Page background uses `dynamicColors.background`
  - EpisodeRow receives `dynamicColors` parameter
  - Alternating card colors use `surfaceLow` (even) / `surfaceHigh` (odd) from extracted palette
  - Falls back to M3 `surfaceContainerLow`/`High` when dynamic theming is disabled
- **Settings toggle**: in Layout subpage → Page layout section → "Dynamic theming" switch

## Decisions made
- **Dynamic theming is opt-in**: default true but can be toggled off in settings
- **Palette extracts 7 colors**: not just one — multiple swatches for surface variants, text, background
- **Color extraction is async**: LaunchedEffect runs on anime load, doesn't block the UI
- **Fallback strategy**: if Palette can't extract a specific swatch, derive from dominant or fall back to M3 defaults
- **Metadata subpage is minimal**: no source names, no detailed explanations — just toggles
- **Layout subpage organized into 5 sections**: each with clear title + one-line description
