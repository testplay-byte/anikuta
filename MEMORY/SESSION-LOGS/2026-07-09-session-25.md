## What happened

### Tasks 1, 2, 5, 6: Episode layout when no thumbnail
**Problem**: When thumbnails were off, the title got cut off (date/pills were beside it in the same container), the episode number position setting was ignored, and the summary/date/pills were crammed in the right-side column.

**Fixes**:
- **EpisodeRowSimple** (no thumbnail, no summary): Complete rewrite
  - Date/pills now BELOW the title (not beside it) — prevents title cutoff
  - Respects `episodeNumberPosition`: "badge" shows badge inside title, "overlay" shows circle
  - Always shows at least the episode number
  - Date/pills use the same pill format as EpisodeRowRich (Surface with dot separators)

- **EpisodeRowRich** (has summary): When no thumbnail
  - Only the title shows in the right-side column
  - Synopsis and date/pills go full-width below the episode number + title row
  - Date "above" and "below" synopsis positions still work

- **EpisodeRowPreview**: Same fixes applied for consistency

### Task 3: Sticky preview spacing
- Increased bottom padding from 4dp to 20dp in all 3 subpages (Display, Layout, Metadata)
- Clear visual separation between the sticky preview and the scrollable settings below

### Task 4: Metadata page preview
- All display toggles forced ON in metadata page preview (regardless of episode display settings)
- Only layout settings apply to the preview
- Removed unused display prefs reads from MetadataSettingsScreen

### Tasks 7, 8: Dynamic theming via MaterialTheme override
**Key insight**: Instead of passing `dynamicColors` to every individual composable, override the entire M3 ColorScheme. This way EVERY composable using `MaterialTheme.colorScheme.*` automatically gets the dynamic colors.

**Implementation**:
- Added `DynamicColorScheme.toM3ColorScheme()` function in DynamicTheming.kt
  - Maps dynamic colors to all M3 color roles (primary, secondary, tertiary, surfaces, etc.)
  - Uses `darkColorScheme()` since the dynamic scheme is always dark-themed
  - Includes luminance calculation for auto black/white text on accent

- DetailScreen: wraps entire content in `MaterialTheme(colorScheme = themedColorScheme)`
  - When dynamic theming is ON: uses `dynamicColors.toM3ColorScheme()`
  - When OFF: uses `MaterialTheme.colorScheme` (no change)

- Elements now automatically themed:
  - Save button (uses primary color)
  - "Show More" button (uses primary color)
  - Source badge (uses secondaryContainer)
  - Episode number, pills, date, summary, title backgrounds (use surfaceContainer variants)
  - Resolving video block (uses surface variants)
  - Video picker sheet (uses surfaceContainerLow, primary)
  - Quality options (uses surfaceContainer, primary)
  - Loading indicators (uses primary, circular progress)
  - Banner gradient (uses background — transitions to dynamic background color)
  - Alternating episode card colors (use surfaceContainerLow/High)

## Decisions made
- **MaterialTheme override is the cleanest approach**: Instead of threading `dynamicColors` through 20+ composable parameters, a single `MaterialTheme(colorScheme = ...)` wrapper themes everything automatically. This is the Compose-idiomatic way.
- **EpisodeRowSimple rewrite**: The old layout put pills beside the title (cutting it off). The new layout puts pills below the title — matching the live preview and EpisodeRowRich format.
- **No-thumbnail layout**: When there's no thumbnail, ALL content (synopsis, date, pills) goes full-width below the episode number + title row. This prevents the right-side column from getting too narrow.
- **Metadata preview forces all display toggles ON**: The metadata page is about fetching data, not about display settings. The preview should always show what the fetched data looks like.
