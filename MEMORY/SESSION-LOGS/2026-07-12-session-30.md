# Session Log — 2026-07-12 (Session 30)

## Goal
Fix subtitle rendering (the main blocker), implement folder selection, reorganize player settings UI.

## What happened

### 1. Sandbox recovery
- Sandbox was completely wiped (project source, SDK, JDK all gone).
- User provided GitHub PAT. Cloned `testplay-byte/anikuta` from GitHub.
- Used GitHub Actions for all builds (not local toolchain).

### 2. Subtitle rendering fix (THE main achievement)
- **Root cause:** The bundled `app/src/main/assets/subfont.ttf` was a GitHub HTML 404 error page (312 KB), not a real TrueType font. libass found it at the correct path but couldn't parse it → "Error opening font" → "can't find selected font provider" → no subtitle text could render.
- **Journey:** 6+ builds of targeted patches failed because all logs were at `warn` level (hid libass errors). Only when verbose logging (`all=v`) was enabled did the real error become visible.
- **Fix:** Replaced the fake HTML file with the real DejaVu Sans TTF (6.4 MB, valid `\x00\x01\x00\x00` signature) from the aniyomi-mpv-lib repo.
- **Documentation:** Full history in `SUBTITLES_FIX.md`.

### 3. Subtitle UI overhaul
- Custom numeric keypad (4×3 grid, +/- buttons, bottom sheet — not popup)
- Color picker (presets + RGBA sliders, live preview, ARGB bug fix)
- Delay stepper (−/value/+ buttons instead of slider)
- Subtitle settings subpage (font, size, colors, position, delay)
- Track name fix (show language instead of ugly .vtt hash filenames)
- Chip-style subtitle track selection (FilterChip + AssistChip)

### 4. Folder selection feature (SAF)
- New `StorageManager` + `StoragePreferences` + `FolderProvider` infrastructure.
- Onboarding step 2 now launches real `ACTION_OPEN_DOCUMENT_TREE` picker.
- `takePersistableUriPermission` for URI survival across reboots.
- Subdirectory structure: `downloads/`, `data/`, `backups/`, `cache/` + `.nomedia`.
- New "Data & Storage" settings category (separate from player settings).
- `StorageSettingsScreen` for changing folder post-onboarding.
- UniFile dependency added (same SAF wrapper aniyomi uses).
- **Documentation:** `STORAGE.md`.

### 5. Player settings reorganization
- Player settings page is now a **hub** with only links to subpages:
  - General (default view, quality sheet display)
  - Playback (speed, hardware decoding, audio language)
  - Subtitles (default mode, preferred language, appearance)
  - Display & Behavior (top bar, auto-hide, gestures, PiP, skip)
  - Episode list (existing subpage)
- New `SelectableOptionCard` component (card-style selector with primary border, not dark blue primaryContainer).
- New `StyledSegmentedRow` component (matches details-page LayoutSettingsScreen design language).
- Episode list settings redesigned to match details-page design language.

### 6. Crash screen fix
- `AnikutaCrashHandler` was posting `startActivity` via `mainHandler.post`, but main-thread crashes kill the Looper before the posted runnable runs.
- Fixed: call `startActivity` synchronously (safe with `FLAG_ACTIVITY_NEW_TASK` from any thread).
- Crash screen now reliably appears on all crashes.

### 7. Font-size application fix
- `applySubtitlePreferences()` was using `setPropertyString` for numeric properties (sub-font-size, sub-scale, etc.). MPV doesn't reliably convert string→number for runtime property sets.
- Fixed: use `setPropertyInt` for integers, `setPropertyDouble` for scale.

### 8. Merge to main
- All work merged from `player-experiment` into `main` (commit `52ed14b`).
- Build verified on main (all steps green).
- `player-experiment` branch preserved for reference.
