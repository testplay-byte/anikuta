DESIGN PLAN · v3

The Video Player
reimagined.

One screen, two modes. YouTube-style minimized & fullscreen — built with Material 3 Expressive and powered by MPV.

01 Architecture

A single screen that morphs between two modes. The video never stops playing during the transition.

Minimized Mode

Video on top, everything else below. Browse episodes while watching.

  • Video adapts to aspect ratio (sides touch edges)
  • Server + version dropdowns below video
  • Episode info + scrollable episodes list
  • Tap video → show/hide controls
  • Maximize button at bottom-right of video
  • Quality + subtitle buttons on video overlay
  • Swipe up → go fullscreen

Fullscreen Mode

Landscape, immersive. All controls overlaid on video. Auto-hide.

  • Video fills entire screen (landscape)
  • Controls auto-hide + tap to toggle
  • Lock button disables all input
  • Full gesture support (full-screen halves)
  • Pinch zoom with magnetic resistance
  • Swipe down → go minimized

05 Pinch Zoom — Continuous with Magnetic Resistance

NOT jumping between values. The zoom is continuous — user can pick any value. But at 20% intervals, there's "resistance" that makes it stick briefly.

🧲 How it actually works (corrected)

NOT jumping between 100→80→60. The zoom is continuous.

If user is at 100% and pinches to zoom out slightly → nothing happens (stuck at 100%). User zooms out a bit more → breaks past the resistance → zoom goes to 99, 98, 97, 96... smoothly.

When zoom reaches 80%, it "sticks" there — user needs to zoom out harder to break past. Then continues: 79, 78, 77...

The resistance zones are at every 20% mark (40%, 60%, 80%, 100%). Between zones, zoom is perfectly smooth and continuous.

08 Episode List Customization

The player's episode list shares the detail page design but has its OWN separate settings — fully independent customization.

Separate Settings

Player episode list preferences are stored separately from the detail page preferences. User can have different layouts in each.

  • Show/hide: number, titles, summaries, thumbnails, dates, audio pills
  • Layout: positions, sizes (same options as detail page)
  • Separate live preview in player settings
  • Alternating colors (same as detail page)

Settings Location

Settings → Player → Episode List
├── Display (show/hide toggles)
├── Layout (positions, sizes)
└── Live Preview
Why separate?
Keeps customization routes free. User might want compact list in player but rich list on detail page.

13 Answered Questions

Confirmed decisions from our discussion.

1. Skip 85 seconds button?
For skipping anime openings. Duration customizable in Settings (default 85s).
2. Lock button behavior?
All controls disabled. Only lock button shown. Auto-hides on tap/inactivity. Tap screen → show lock → tap lock → unlock.
3. Server selection in minimized mode?
Proper dropdowns for server + audio version. Quality + subtitle selection are on the video overlay itself.
4. Replace current full-screen player?
Yes, replace it. The new combined player's fullscreen mode handles everything.
5. Performance testing?
Will consult/highlight performance concerns as they arise. Release build testing in Phase 6.
6. Notifications?
After EACH step (not just each phase). ~25-30 notifications across all phases.
